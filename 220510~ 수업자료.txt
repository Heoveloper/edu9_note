변수: 값을 저장하기 위해 메모리 공간에 붙여진 이름.

변수의 생명주기 (생성~소멸): 변수를 선언한 블럭 내에서만 접근이 가능하다.

연산식: 피연산자 연산자 피연산자

지역변수: 메소드 내에 선언된 변수
		 초기화 이후에 사용할 수 있다.
		 초기화 : 변수가 선언된 이후 최초로 값을 할당하는 표현식

매개변수 (파라미터, 인수): 메소드 선언부에서 메소드 외부에서 입력받기 위해 선언된 변수
					 메소드 내에서 지역변수처럼 사용됨.
		 
타입: 변수의 값의 종류와 범위(메모리)를 결정함.
     기본형(8개), 참조형 2가지가 존재
	 
연산자: 산술 > 비교 > 논리 > 대입
	   증감연산자(i++, ++i, i--, --i);
	   부호연산자(+, -);
	   문자열연결연산자(+): 피연산자 중 하나라도 문자열이면 연결하여 문자열로 반환
	   
실행 클래스: 메인 메소드(psvm)를 포함하고 있는 클래스.

라이브러리 클래스: 메인 메소드를 포함하고 있지 않는 클래스.
	- 종류 1. 자바 언어가 기본적으로 제공해주는 클래스 (표준 클래스, 빌트인 클래스, 표준 라이브러리)
		  2. 개발자가 필요에 의해 생성해서 사용하는 클래스
		  3. 제 3자가 제공하는 라이브러리 (클래스의 모음) (3th party library, 오픈 라이브러리)


배열: 하나의 변수로 여러 개의 값을 메모리 연속공간에 저장할 수 있는 자료구조

배열요소: 배열에 저장된 값 하나하나를 뜻한다.

배열요소 접근방법: 배열변수이름[인덱스]

배열선언: int[] 변수이름; or int 변수이름[];

배열생성: 변수이름 = new int[요소의 크기];

null: 참조형 변수만이 가질 수 있는 값.
	  힙메모리에 아무 객체도 참조하지 않음.
	  
	  
표현식: 하나의 값으로 평가되는 식(값, 변수, 식, 메소드호출)

final: 최종, 마지막의
    final + variable: 초기화 이후 재할당 불가능하다.
    final + class: 최종개념으로 하위개념이 올 수 없다.
    final + method: 상위개념의 메소드를 하위개념에서 재정의가 불가능하다.
