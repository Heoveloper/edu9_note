-퀴즈-
1. 예외처리란?
	예외가 발생했을 때 어떤 동작으로 처리할지 설정하는 것

2. checked exception & unchecked exception 차이?
	꼭 처리해야 하는 예외와 그렇지 않은 예외

3. 예외처리 하는 방법 2가지?
	1) try~catch~finally로 감싼다
	2) 위임해준다

4. 오버로딩(overloading)이란?
	동일클래스 내에서 같은 이름의 메소드나 생성자를 중복하여 정의하는 것

5. 오버로딩이 성립하기 위한 전제조건?
	1) 메소드명이 동일해야 한다
	2) 매개변수 개수나 타입이 달라야 한다

6. 오버라이딩(overriding)이란?
	부모클래스에서 정의된 메소드를 자식클래스에서 재정의해서 사용하는 것

7. 오버라이딩이 성립하기 위한 전제조건?
	1) 메소드 선언부가 동일해야 한다
	2) 인스턴스 메소드만 가능 (private메소드, final 불가능)
	3) 접근제한자는 동일하거나 범위가 더 넓어야 한다

8. 생성자의 역할?
	인스턴스 생성과 초기화

9. class란?
	객체를 만들기 위한 설계도

10. interface란?
	내부의 모든 필드가 public static final로 정의되고,
	static, default 메소드 이외의 모든 메소드는 public abstract로 정의된
	객체지향 프로그래밍 요소

11. final 키워드의 3가지(변수, 클래스, 메소드) 쓰임?
	final변수) 초기화 이후 재할당 불가
	final클래스) 최종 클래스로 하위 클래스가 올 수 없다
	final메소드) 상위 메소드를 하위 메소드에서 재정의할 수 없다

12. 반환문(return)의 용도?
	현재 메소드 종료 후 호출한 곳으로 제어 이동
	return 표현식(값, 변수, 식, 함수 호출): 현재 메소드 종료 후 반환값을 호출한 곳으로 전달

13. 매개변수란?
	메소드 외부에서 입력받기 위해 메소드 선언부에서 선언된 변수
	메소드 내에서 지역변수처럼 사용된다

14. 지역변수란?
	메소드 내에 선언된 변수
	초기화(변수 선언 이후 최초로 값을 할당) 이후 사용할 수 있다

15. 추상클래스란?
	추상메소드를 1개 이상 포함하고 있는 클래스

16. 추상메소드란?
	메소드의 바디가 미완성인 메소드

17. 접근제한자 4가지에 대해 설명하시오
	public) 어디서든 접근 가능
	protected) 동일패키지나 상속하는 다른 패키지에서만 접근 가능
	default) 동일패키지에서만 접근 가능
	private) 동일클래스에서만 접근 가능

18. 인스턴스 멤버와 정적 멤버의 차이?
	인스턴스 멤버는 인스턴스 생성 후 접근 가능하지만 정적 멤버는 인스턴스 생성 없이 클래스명.멤버로 접근 가능
	인스턴스 속성값은 각자 다르고 정적 속성은 공통이다

19. 익명클래스를 생성하는 2가지 방법?
	1) 부모클래스를 상속해서 오버라이딩
	2) 인터페이스명을 사용해서 정의

20. 익명클래스를 사용하는 이유?
	일시적으로 한번만 사용되어야 하는 객체일 경우 사용
